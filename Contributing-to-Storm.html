Contributing

Getting started with contributing

issue tracker にあるいくつかの問題には「Newbie」というラベルが付けられています。 もしStormにコントリビューションすることに興味があるものの、どこからはじめるべきか分からなければ、これらは手始めとするのに良い問題です。これらの問題は、コードベースの独立した部分についてのみ学習する必要があり、比較的少ない作業量しか必要としないため、コードベースの学習にとても良い方法です。

Learning the codebase

wikiにある 実装に関するドキュメント のセクションでは、コードベースの詳細なウォークスルーを提供しています。 コードベースを理解するために、これらのドキュメントを読むことを強くお勧めします。

Contribution process

Stormのコードベースへの貢献は、GitHub に対するプルリクエストとして送られる必要があります。 プルリクエストに問題がある場合は、GitHubのコメント機能を使用して改善を繰り返すことができます。

小さなパッチの場合は、プルリクエストを直接提出してください。 より大きなコントリビューションを行う場合、以下のプロセスを使用してください。 このプロセスの背景、無駄な作業を防止するとともに、早い段階で設計上の問題を捉えることです。

  1. issue tracker に存在しない問題については、起票してください
  2. 問題を実装するための方法についてコメントしてください。 あなたが作業する予定のコードベースの各部分と、それらがどのように結合されるかを説明してください
  3. Stormコミッタ達は、あなたが正しい軌道に乗っていることを確認するために、設計上の観点であなたと繰り返し議論します
  4. 問題解決方法を実装し、プルリクエストを送信し、それらを繰り返します

Modules built on top of Storm

Stormのコアに実装するには適していないStormの上に構築されたモジュール（spouts, boltsなど）は、あなた自身のプロジェクトとして、または@stormprocessorの一部として実装することができます。@stormprocessorに参加するには、自分のプロジェクトを自身のGithubに入れて、メーリングリストに@stormprocessorの一部にするという電子メールを送ってください。そうすると、コミュニティは@stormprocessorに取り込むことが有益であるか議論することができます。 それから、あなたは@stormprocessorのorganizationに追加され、そこでプロジェクトをメンテナンスすることができます。 @stormprocessorにモジュールをホストする利点は、潜在的なユーザーがプロジェクトを簡単に見つけることができることです。

About Storm

Storm integrates with any queueing system and any database system. Storm's spout abstraction makes it easy to integrate a new queuing system. Likewise, integrating Storm with database systems is easy.
